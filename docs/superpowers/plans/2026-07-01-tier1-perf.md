# Tier 1 Performance Optimizations Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Ship the audit's Tier 1 performance wins — WebP images, fewer re-renders during analysis, a cached IndexedDB connection, and cheaper filtering — with **zero visual or behavioral change**.

**Architecture:** Four independent optimization areas: (1) convert raster assets to WebP + update imports; (2) narrow Zustand subscriptions so analysis updates stop cascading through the tree; (3) cache the IndexedDB connection + bulk-check keys; (4) stop rebuilding the feature normalizer on every slider tick + memoize hot-path computations.

### Task 4: Cache the IndexedDB connection + bulk-check keys before background analysis

`featureCache.ts` opens a fresh `IDBDatabase` on every read/write, and `useBackgroundAnalysis.ts:33-37` serially `await getFeatures(t.id)` over the **entire library** (up to 3000 sequential connection-opens + round-trips) before analysis starts. Cache the connection in a module-level promise and add a bulk key check.

**Files:**
- Modify: `src/services/featureCache.ts`
- Modify: `src/hooks/useBackgroundAnalysis.ts:29-37`

**Interfaces:**
- Produces: `getAllKeys(): Promise<Set<string>>` exported from `featureCache.ts`.

- [ ] **Step 1: Cache the connection in `featureCache.ts`**

Replace the `openDB` function so the connection is opened once and reused:
```ts
let _dbPromise: Promise<IDBDatabase> | null = null

function openDB(): Promise<IDBDatabase> {
  if (_dbPromise) return _dbPromise
  _dbPromise = new Promise((resolve, reject) => {
    const req = indexedDB.open(DB_NAME, VERSION)
    req.onupgradeneeded = () => {
      const db = req.result
      if (db.objectStoreNames.contains(STORE)) db.deleteObjectStore(STORE)
      db.createObjectStore(STORE, { keyPath: 'id' })
    }
    req.onsuccess = () => resolve(req.result)
    req.onerror = () => reject(req.error)
  })
  return _dbPromise
}
```
(The three exported functions `getFeatures`/`setFeatures`/`getAllFeatures` keep calling `await openDB()` — they now share one connection.)

- [ ] **Step 2: Add a bulk `getAllKeys` to `featureCache.ts`**

Append:
```ts
export async function getAllKeys(): Promise<Set<string>> {
  const db = await openDB()
  return new Promise((resolve, reject) => {
    const tx = db.transaction(STORE, 'readonly')
    const req = tx.objectStore(STORE).getAllKeys()
    req.onsuccess = () => resolve(new Set(req.result as string[]))
    req.onerror = () => reject(req.error)
  })
}
```

- [ ] **Step 3: Use the bulk check in `useBackgroundAnalysis.ts`**
