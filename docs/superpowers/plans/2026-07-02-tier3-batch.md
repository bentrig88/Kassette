### Task 1: Unbiased shuffle + remove the 3000-track cap (`appleMusic.ts`)

**Files:** Modify `src/services/appleMusic.ts`.

- [ ] **Step 1: Replace the biased shuffle with Fisher–Yates**

In `loadCassetteQueue`, replace:
```ts
  const shuffled = [...cassette.tracks].sort(() => Math.random() - 0.5)
```
with a proper Fisher–Yates:
```ts
  const shuffled = [...cassette.tracks]
  for (let i = shuffled.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1))
    ;[shuffled[i], shuffled[j]] = [shuffled[j], shuffled[i]]
  }
```

- [ ] **Step 2: Remove the 3000-track fetch cap**

In `fetchLibraryTracks`, delete the safety-cap block:
```ts
    if (tracks.length >= 3000) {
      hasMore = false
    }
```
(and its `// Safety limit` comment if present). The loop already terminates when `response.data.next` is falsy or a page returns fewer than `limit` items, so full pagination is preserved.

### Task 2: Tighten the slider-unlock threshold (`PlaylistController.tsx`)

The sliders currently unlock at `min(5, upcoming.length)` analyzed upcoming tracks, which can enable them when the library-wide normalizer has too little data to rank anything (percentiles collapse to a neutral 50, so the sliders feel dead). Add a library-wide floor.

**Files:** Modify `src/components/PlaylistController.tsx`.

- [ ] **Step 1: Add a library-wide analyzed floor to `enoughData`**

Replace:
```ts
  const enoughData = analyzedUpcoming >= Math.min(5, upcoming.length)
```
with:
```ts
  // Also require ≥5 analyzed tracks library-wide so the percentile normalizer
  // can actually rank (with ≤1 it returns a flat 50 and the sliders do nothing).
  const enoughData = analyzedUpcoming >= Math.min(5, upcoming.length) && featuresMap.size >= 5
```
(`featuresMap` is already subscribed in this component.)

Run: `npm run build && npm run lint` → both exit 0.

### Task 4: Empty-library guard (`App.tsx`)

If the library matches zero genres, `cassettes.length === 0`, the player branch never renders and — once the loading overlay unmounts — the screen goes blank. Add an empty-state branch. Also extract the sign-out handler so both the player and the empty-state reuse it.

**Files:** Modify `src/App.tsx`.

- [ ] **Step 1: Extract a `handleSignOut` callback**

Above the `let screen` block, add (matching the existing sign-out logic in the player branch):
```tsx
  const handleSignOut = async () => {
    try {
      getMusicKitInstance().stop()
      await getMusicKitInstance().unauthorize()
    } catch {/* */}
    setAuthenticated(false)
    setCassettes([])
    setIntroDone(false)
    setLoadingComplete(false)
    setTracksSoFar([])
  }
```
Then replace the inline `onClick={async () => { … }}` on the player branch's Sign out button with `onClick={handleSignOut}`.

- [ ] **Step 2: Add the empty-library branch**

After the `else if (cassettes.length > 0) { … }` branch, add:
```tsx
  } else if (loadingComplete) {
    // Authed, load finished, but no track matched any genre → no cassettes.
    screen = (
      <div className="loading-screen">
        <div className="loading-card">
          <div className="loading-title">No cassettes to show</div>
          <p style={{ marginBottom: '1rem' }}>
            None of the tracks in your library matched a genre we recognize.
          </p>
          <button className="auth-button" onClick={handleSignOut}>Sign out</button>
        </div>
      </div>
    )
  }
```
(At this point `!isAuthenticated`, `error`, and `cassettes.length > 0` are all false, so `loadingComplete` here means: authed, no error, load done, zero cassettes. The `LoadingScreen` overlay has already unmounted because `loadingComplete` is true, so this card is visible.)

- [ ] **Step 3: Verify build + lint**

Run: `npm run build && npm run lint` → both exit 0.

- [ ] **Step 4: Commit**

```bash
git add src/App.tsx
git commit -m "Add empty-library guard (empty-state instead of blank screen)"
```
